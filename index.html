How It Works

This is a client-side LLM token counter. Nothing you type is sent anywhere - all processing happens in your browser. It's useful for getting a quick ballpark before you run something expensive, or for checking if your prompt will fit in a model's context window.

The approximation (~4 characters per token) holds reasonably well for English text. Code tends to tokenize differently since punctuation, brackets, and operators often get their own tokens, so we switch to ~3.5 chars/token when code is detected. For anything where exact counts matter, use the model's official tokenizer.

Common Questions

How accurate is this token counter?

Typically within 10-15% for English prose. The 4 chars/token rule is a well-known approximation that works across GPT, Claude, and Gemini. For code, punctuation-heavy text, or non-English languages, accuracy drops. For exact counts you need the model's tokenizer.

Which Claude model has the cheapest API cost?

Claude 3.5 Haiku at $0.80 input / $4.00 output per 1M tokens. For OpenAI, GPT-4o mini ($0.15/$0.60) is the budget option. Gemini 1.5 Flash ($0.075/$0.30) and Gemini 2.0 Flash ($0.10/$0.40) are currently the cheapest options with large context windows.

Does this tool send my text anywhere?

No. Everything runs in your browser. There's no backend, no analytics collecting your text, no API calls. You can load the page, disconnect from the internet, and it still works.
